Overview

This repository contains a collection of simple CGI files. The two main purposes of this repositiory are:
  • To provide a set of CGI programs to illustrate the CGI protocol.
    Because of their simplicity, they are easier to understand. These programs can also be used to validate a user's enviornment. It is expect that this programs will be installed on a simple webserver that supports generic CGI programs.
  • To provide the specification of a simple docker image.
    The resulting container will include each of these CGI programs. Then these CGI programs can be used to validate and demonstrate the function of the docker-cgi project.

Directory of CGI Files

  • noop.cgi: The most trivial of all CGI programs, it does nothing.
  • echo-request.cgi: A CGI program that emits the components of the query string and the request body.
  • emit-cgi-env.cgi: A CGI program the emits the environment variables defined by the CGI protocol.
  • emit-env.cgi: A CGI program the emits the environment variables provided by the Web-server, including the CGI variables.
  • here.cgi: A CGI program that emits simple HTML document contained within a here document.
  • cat.cgi: A CGI program "cat"s a file based upon the query string.
    E.g., the "cat.cgi?cat.cgi" "cat"s out the "cat.cgi" program.

Source of CGI Files

  • noop.cgi: Source via cat.cgi?noop.cgi
  • echo-request.cgi: Source via cat.cgi?echo-request.cgi
  • emit-env.cgi: Source via cat.cgi?emit-env.cgi
  • emit-cgi-env.cgi: Source via cat.cgi?emit-cgi-env.cgi
  • here.cgi: Source via cat.cgi?here.cgi
  • cat.cgi: Source via cat.cgi?cat.cgi

Container-based Access

Each of the above CGI programs have also been placed into a docker container. These CGI programs are made available by use of the docker-cgi project.

The specification of the docker image is provided by this dockerfile.

Given the current state of this project, an Apache Handler is used to access these programs. This handler is invoked for any file with the extension of ".docker-cgi".
